All about Tailwind CSS:

What is Tailwind CSS?

Tailwind CSS is known as a utility-first CSS framework. A framework is a pre-defined set of tools and guidelines that helps you develop software more efficiently, quickly, and in a more structured way. A utility-first framework provides small, reusable classes — called utility classes — that you apply directly in your HTML to style elements.

Tailwind CSS focuses on providing utilities to help you build interfaces without spending too much time writing custom CSS stylesheets. Unlike traditional frameworks, a utility-first approach means that each class applies a specific change to the appearance of the element. To achieve the desired result, the developer combines multiple Tailwind classes directly in the HTML.

What does Tailwind do?

Tailwind allows you to quickly build design prototypes and experiment with different looks and layouts. As your design evolves, Tailwind CSS lets you organize and group utility classes into custom CSS, helping simplify your HTML markup once you've finalized the design.

Why use Tailwind?

  • No More Writing Your Own CSS for Basics: Tailwind offers layout, spacing, typography, and other utility classes, so you hardly ever need to create your own CSS.
  • Rapid Prototyping: Ideal for quick iterations and MVPs since you can build and modify designs right in your HTML.
  • Component-Friendly: Promotes consistent, reusable user interface elements with little stylistic costs.
  • Extremely Customizable: You may customize Tailwind configuration file to fit your brand or design system by defining your own colour scheme, spacing scale, breakpoints, and more.

Where should I use Tailwind?

  • Startups & Large-Scale Apps: Tailwind grows with your project, from rapid MVPs to apps that are ready for production.
  • Admin dashboards: Perfect for quickly creating UIs that are clear, responsive, and useful.
  • Landing pages and marketing websites: Easily build attractive layouts with integrated design tokens and responsiveness.
  • Portfolios and blogs: Developers who want complete control over design without having to write a ton of CSS would love it.

Tailwind Examples

Basic Text Stylings

Tailwind CSS makes it easy to style text directly in your HTML using utility classes. You can adjust font weight with classes like font-bold or font-thin, add styles such as italic or underline, and customize decorations with options like decoration-wavy and color utilities. Tailwind also provides font family utilities, including font-sans, font-serif, and font-mono, so you can quickly switch between different typefaces without writing custom CSS.

You can also control text size using the text-* utility classes. Tailwind's default text size scale includes options like text-xs, text-base, text-xl, and up to text-9xl, making it easy to consistently scale headings, body text, captions, and more. These classes not only adjust the font size but also automatically manage line height for improved readability.

See the Pen Tailwind Basic stylings by MelissaGoon (@MelissaGoon) on CodePen.

Colours in Tailwind

Tailwind provides a diverse default colour palette designed to cover a wide range of design needs. For example, bg-blue-500 for a medium blue background, or text-gray-800 for dark gray text. Custom values can also be used for colours, for example: bg-[#facc15].

You can apply these colour utilities to backgrounds, text, borders, rings, shadows, and even gradients, giving you flexible and consistent control over the visual style of your project.

All Tailwind's colours come in 11 steps, with the lightest being 50 and the darkest being 950.

See the Pen Tailwind Default Colors by MelissaGoon (@MelissaGoon) on CodePen.

Tailwind and the CSS Box Model

In Tailwind CSS, you can control every part of the CSS box model using utility classes. Set padding and margin with shorthand classes like p-* and m-*, where the values follow Tailwind’s default spacing scale, typically ranging from 0 to 96. Auto and custom values can also be used, for example: m-auto and p-[1.1rem].

You can also target specific sides or directions using the utility classes. For example, pt-* for padding-top, pb-* for padding-bottom, pl-* for padding-left and pr-* for padding-right. Furthermore, handy classes such as px-* and py-* allow you to manipulate both the left and right or top and bottom paddings respectively. This makes it easy to fine-tune spacing without custom CSS.

See the Pen Tailwind Box Model by MelissaGoon (@MelissaGoon) on CodePen.

Apply borders with utilities like border, and customize border width, color, and radius as needed.

Control an element’s width and height using classes such as w-*, h-*, or percentage-based values like w-1/2. By default Tailwind has border-box active. Tailwind also provides min and max classes for setting minimum and maximum width and height, such as min-w-* or max-h-*. For layout flexibility, you can use w-full or h-full to make an element fill its parent container, or w-screen and h-screen to stretch an element to the full width or height of the viewport.

Flex & Grid

Tailwind CSS makes it simple to build responsive, flexible layouts using utility classes for both Flexbox and CSS Grid.

To create a flex container, use the flex class on a parent element. You can then control the alignment of child elements with utilities like justify-center (equivalent to justify-content:center ) and items-baseline ( align-items:baseline ).

CSS Grid in Tailwind works by applying the grid class to a parent container, then defining the number of columns using grid-cols-* and spacing between grid items with gap-*. You can control how individual grid items span rows or columns using classes like col-span-2, row-span-2, or set their position within the grid using classes like col-start-*, col-end-*, and row-start-*.

Pseudo-Classes & Responsive Styling

Tailwind CSS makes it easy to apply styles based on user interaction and device size using utility-friendly pseudo-class variants and responsive prefixes. Pseudo-classes like hover:, focus:, active:, and disabled: can be added in front of any utility class to change its behavior in specific states. For example, hover:bg-blue-500 changes the background color when a user hovers over an element, and focus:ring-2 adds a ring outline when the element receives focus.

Responsive styling in Tailwind is mobile first, appying styles at specific screen width breakpoints. The default breakpoints are sm (40rem), md (48rem), lg (64rem), xl (80rem), and 2xl (96rem). You can combine these with any utility class to apply different styles at different viewport sizes.

Tailwind also provides built-in support for dark mode, making it simple to create themes that adapt to the user’s system preference or a custom trigger. By default, you can prefix any utility class with dark: to apply it when dark mode is active.

Both pseudo-class variants and responsive prefixes can be combined in a single class name, for example: hover:lg:text-pink-500.

See the Pen Pseudo-Classes & Responsive Styling by MelissaGoon (@MelissaGoon) on CodePen.

Built-In Animations

Tailwind CSS includes a set of simple, built-in animations that you can apply to elements using utility classes. They include animate-spin, animate-ping, animate-bounce, animate-pulse and animate-none used to override any other animations.

All of these animations are applied by simply adding the class to an element, and you can customize their speed, delay, or other behaviors by extending your tailwind.config.js if needed.

See the Pen Tailwind Built In Animations by MelissaGoon (@MelissaGoon) on CodePen.

Some Decorative Utilities

In Tailwind CSS, you can easily add shadows to elements using the shadow utility classes. These vary in size, with options like shadow-sm, shadow-md, shadow-lg, and shadow-2xl.

Alongside box shadows, Tailwind offers ring utilities for creating outlines around elements. Unlike borders, rings sit outside the element’s border and are often useful for focus states or emphasis. You can control ring thickness with classes like ring-*, and adjust their color and offset using classes like ring-blue-500 and ring-offset-*.

Tailwind also simplifies working with gradients using utility classes like bg-gradient-to-r, bg-gradient-to-b, and more, which set the gradient direction. You can define starting, middle, and ending colors using from-*, via-*, and to-* classes. For example, a smooth horizontal gradient from blue to pink would use bg-gradient-to-r from-blue-500 to-pink-500. This makes creating custom, responsive gradients quick and consistent without writing custom CSS rules.

See the Pen Decorative Utilities by MelissaGoon (@MelissaGoon) on CodePen.

Accessibility in Tailwind

Tailwind CSS includes several built-in utilities to help improve the accessibility of your projects. For example, it includes a sr-only class, which visually hides content but keeps it available for screen readers. This is helpful for adding extra descriptive text, instructions, or labels that assistive technologies can access without displaying them on the screen. To make hidden content visible again, you can use not-sr-only.

Tailwind also respects user preferences for reduced motion by offering the motion-reduce variant. You can use this to disable or simplify animations and transitions for users who have enabled "Reduce Motion" in their operating system settings. For example: motion-reduce:animate-none. You can also use the motion-safe variant to mark certain animations as safe for users who prefer reduced motion.
